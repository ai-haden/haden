# Agent Working Rules

These rules apply to all future assistant work in this repository.

## Change Approval

- Before making any code changes, the assistant must:
  - propose the intended changes, and
  - wait for explicit user approval.
- No file edits are allowed before approval, except when the user explicitly asks for immediate implementation.

## Changelog Requirement

- Every approved code fix must be recorded in `CHANGELOG.md`.
- The changelog entry must include:
  - affected file path(s),
  - a short description of the bug,
  - a short description of the behavior change.
